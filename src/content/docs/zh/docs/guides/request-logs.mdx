在左侧 **User → Request Logs** 打开请求日志。这里记录的是 **经 Proxy 的真实推理调用**：时间、密钥、用户、模型、路由组、状态等信息，用于排障与用量核对。

<Aside type="tip" title="前置">
	需要已有流量经过 **Proxy**。[Playground](/zh/docs/features/playground/) 的试跑方式**不会**出现在本日志中；若要产生与线上一致的记录，请用真实密钥调用 Proxy 或使用 [Simulator](/zh/docs/features/simulator/)。
</Aside>

## 在管理台里要做什么

<Steps>
1. 打开 **Request Logs**。
1. 使用时间范围、状态、模型、用户邮箱、API Key、供应商、路由组、协议等条件缩小范围（以页面筛选项为准）。
1. 点击展开某一行，查看计费核对摘要、入口请求体、上游请求体、用量原始 JSON 等（可复制）。
1. 核对金额与时间时，请与你在 **系统配置** 中设置的业务时区、计费币种保持一致理解。
</Steps>

## 与计费展示的关系

展开区域可能包含与扣费相关的多段 JSON 摘要；若需精确公式或与账单对账，请结合主仓库文档或内部财务口径。
