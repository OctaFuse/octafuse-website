在左侧 **User → API Keys** 打开密钥页。**API Key**（`sk-…`）是应用调用网关 **Proxy** 时的凭证：请求头使用 **`Authorization: Bearer sk-…`**。

列表中会显示密钥状态、最近使用时间，以及与其所属用户相关的 **预算只读摘要**（预算归属在用户侧，便于一眼核对）。创建时可选择 **挂到已有用户**，或通过 **外部身份** 由系统在后台解析或创建对应用户（以创建向导为准）。

<Aside type="tip" title="前置">
	建议已存在至少一个 [用户](/zh/docs/guides/users/)，或在创建密钥时选用外部身份模式。
</Aside>

## 在管理台里要做什么

<Steps>
1. 打开 **API Keys**。
1. **新建**：按向导选择创建方式 → 选定或填写用户信息 → 填写密钥显示名等 → 保存后**立即复制**完整 `sk-…`（界面通常会提示「仅本次可见」）。
1. **日常运维**：在列表中查看状态与摘要；需要时使用编辑、吊销、删除等操作。
1. **自检**：用你的 Proxy 根地址调用 **`GET /v1/me`**，携带 `Bearer sk-…`，确认密钥可用且预算展示符合预期。
</Steps>

```bash
curl -sS "http://localhost:8787/v1/me" \
  -H "Authorization: Bearer sk-your-api-key"
```

将示例中的地址与密钥换成你的实际环境。

## 自动化

服务端批量开通或轮换密钥，请使用管理接口，索引见 [GitHub 技术参考](/zh/docs/github-reference/)。
