在左侧 **User → Audit Logs** 打开审计页。这里的记录关注 **「账户与预算侧发生了什么」**：例如预算调整、用量扣减、用户或密钥的创建与变更、管理台对用户/密钥的修改等，并尽量标注关联的密钥或请求上下文。

它**不是**「在管理台里每一次改供应商、改路由菜单点击」的完整配置审计台账；若你需要配置变更追责，请结合变更管理流程或另行接入运维审计。

<Aside type="tip" title="与请求日志的区别">
	**请求日志**：每一次推理 HTTP 调用轨迹。**审计日志**：与 **用户预算与身份/密钥生命周期** 相关的变更与扣减记录。
</Aside>

## 在管理台里要做什么

<Steps>
1. 打开 **Audit Logs**。
1. 用时间范围、用户邮箱、密钥、事件类型、操作者类型、原因码、来源等条件筛选（以页面为准）。
1. 展开行查看预算计划说明、用户快照差异、变更详情等（界面会省略与列表重复的字段）。
1. 若只关心某一用户：也可从 [用户管理](/zh/docs/guides/users/) 进入该用户详情页查看相关片段。
</Steps>
