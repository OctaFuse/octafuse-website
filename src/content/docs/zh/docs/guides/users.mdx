OctaFuse 中的 **用户** 是网关内的逻辑主体：可拥有 **多个 API Key**、独立 **预算与周期**，并出现在 **请求日志 / 审计日志** 中。常见对接方式是与门户账号或 `externalSystem + externalUserId` 对齐。

<Aside type="tip" title="前置">
	已完成 [快速开始](/zh/docs/quick-start/) 并能登录 Admin。
</Aside>

## 在管理台里要做什么

<Steps>
1. 打开 **Users / 用户** 列表页。
1. **创建用户**：录入 **邮箱**（或组织内唯一标识）、可选 **外部系统 + 外部用户 ID**（便于与 SaaS 账号幂等对齐）。
1. **状态**：可按需启用 / 停用用户；停用后其下 Key 通常应不再用于生产调用（以实际策略为准）。
1. **预算与周期**：若界面将预算放在用户维度，在此设置 **周期与上限**；若你的部署将预算放在 Key 维度，请以界面为准，并阅读 [API Key 管理](/zh/docs/guides/api-keys/)。
1. **详情与审计**：用户详情页可查看关联 Key、预算重置记录等；细粒度变更见 [审计日志](/zh/docs/guides/audit-logs/)。
</Steps>

## 与门户集成

上层系统在用户注册或登录成功后，可调用 **Admin API** 按外部身份 **查找或创建** 网关用户，再为其签发 Key。接口说明见 [GitHub 技术参考](/zh/docs/github-reference/)。
