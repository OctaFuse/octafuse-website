在左侧 **User → Users** 打开用户列表。**用户**是网关内的账户主体：用于挂载 API 密钥、承载 **预算与结算周期**，并出现在请求与审计相关视图中。支持填写邮箱、可选的外部系统标识，以及预算相关字段。

用户 **详情页**（点击列表中的用户进入）通常包含：预算计划、关联的 **API Keys**、与该用户相关的 **请求日志** 与 **审计类记录** 入口（以当前界面布局为准）。

<Aside type="tip" title="前置">
	已完成 [快速开始](/zh/docs/quick-start/) 并能登录管理台。
</Aside>

## 在管理台里要做什么

<Steps>
1. 在 **Users** 列表中按邮箱、外部身份、状态等条件筛选，浏览分页结果。
1. **新建用户**：在表单中填写邮箱、可选的外部系统与用户编号、预算上限/基准、结算周期等；可按需填写 **metadata**（一般为 JSON，用于业务自定义标记）。
1. **查看或编辑详情**：从列表进入用户详情，核对预算、关联密钥与日志。
1. **发放密钥**：在 [API Key 管理](/zh/docs/guides/api-keys/) 中为该用户创建密钥，或使用「外部身份」方式在创建密钥时一并解析用户。
</Steps>

## 与业务门户配合

你的服务端可在用户注册成功后，通过管理接口创建或更新网关用户并下发密钥；环境变量与调用边界见 [系统集成](/zh/docs/guides/integration/)，接口索引见 [GitHub 技术参考](/zh/docs/github-reference/)。
