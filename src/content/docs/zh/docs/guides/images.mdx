Gateway 对外提供 OpenAI 兼容的 **`POST /v1/images/generations`** 与 **`POST /v1/images/edits`**（multipart）。鉴权与 Chat 相同（用户 `sk-…`）；模型须挂 **OpenAI 协议** 的 active 路由，并在目录里配置 Image 计价。

<Aside type="tip" title="前置">
	请先完成 [供应商](/zh/docs/guides/providers/)（含 Images generations 端点）与 [模型](/zh/docs/guides/models/)。字段级契约见 [GitHub 技术参考 · image-models](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/developers/reference/image-models.md)。
</Aside>

## 在管理台里要做什么

<Steps>
1. **Provider**：为 OpenAI / 兼容上游配置 Key；`endpoints.openai.base` 可派生 `…/images/generations`，或写完整 `images.generations` URL。Seedream 等无 OpenAI edits 形态的上游 **不要** 写 `base`（避免派生死链 edits）。
1. **Models**：导入或手建图片生成预设（`output` 含 `image`）。确认 `image_billing_mode`：
	- **`token`**：按上游 usage 的 image / text token 分项计价（如 GPT Image）。
	- **`per_image`**：按张目录价（如 Seedream），可选参考图 input 单价。
1. **Routes**：为该模型添加 **OpenAI** 协议、**启用** 状态的路由。
1. **验收**：Playground 试单条路由出图（不计用户账单）；Simulator 走真实 Proxy → 在 [请求日志](/zh/docs/guides/request-logs/) 核对 `billing_kind` / image count 与扣费。
</Steps>

## 与模型列表的关系

- 默认 **`GET /v1/models`**（需用户 Key）只返回 LLM，**不含**纯图片生成模型；可传 `kind=image` / `kind=all`。
- 门户 discovery 用公开 **`GET /catalog/models`**（无需 Key），与 `/v1/models` 分工不同。

## 客户端最小示例

```bash
curl -sS "$GATEWAY_URL/v1/images/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-image-2","prompt":"a watercolor fox","size":"1024x1024"}'
```
