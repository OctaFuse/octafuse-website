**系统配置** 影响整站行为，例如 **Master Key**（调用 `/api/admin/*` 的 Bearer）、部分全局策略等。误改可能导致自动化脚本无法管理网关，或安全风险上升。

<Aside type="caution" title="权限">
	仅让必要人员拥有修改权限；生产环境务必使用强随机 **Master Key** 并安全分发。
</Aside>

## 在管理台里要做什么

<Steps>
1. 打开 **Settings / 系统设置**（或 **Config**）页面。
1. **查看当前 Master Key 策略**：本地 Docker 种子常为 `sk-dev-admin-key` — **生产必须轮换**。
1. **修改后同步下游**：所有持有 **`GATEWAY_MASTER_KEY`** 的门户、脚本需同步更新。
1. **其他全局项**：按组织规范逐项确认含义后再保存；不确定时先查阅 [GitHub 技术参考](/zh/docs/github-reference/) 或主仓运维文档。
</Steps>

## 与 Admin API 的关系

自动化脚本使用 **`Authorization: Bearer <MASTER_KEY>`** 调用 **`{GATEWAY_MASTER_URL}/api/admin/*`**。完整端点列表见 [GitHub 技术参考](/zh/docs/github-reference/)。
