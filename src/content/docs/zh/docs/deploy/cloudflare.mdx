该形态下：**Proxy** 与 **Admin** 各为一个 **Worker**，**共享同一 D1**（绑定名 **`DB`**）。这是 OctaFuse 的**默认生产路径**——个人与小流量通常可在 Workers / D1 免费额度内完成部署与日常使用。

<Aside type="tip" title="外部用户最快上云">
	仓库根执行 `npm run bootstrap:cloudflare` 即可创建/复用 D1、跑迁移、部署双 Worker，并写入 Admin Secret。完整说明见 [gateway Cloudflare 快速部署](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/operators/deployment/cloudflare-quickstart.md)。
</Aside>

<Aside type="note" title="权威细节">
	本页是面向部署者的常用路径摘要。Workers Builds、多实例、错误 **10181** 等完整运维说明见 [GitHub 技术参考](/zh/docs/github-reference/) 中的 **Cloudflare 部署**。
</Aside>

## 一键 bootstrap（推荐）

```bash
git clone https://github.com/OctaFuse/octafuse-gateway.git
cd octafuse-gateway
npm install
npx wrangler login
npm run bootstrap:cloudflare
```

脚本会：创建或复用 D1 → 远程迁移 → 部署 Proxy / Admin → 写入 `ADMIN_PASSWORD` Secret → 打印下游 `GATEWAY_*` 提示。

后续发版：

```bash
npm run deploy:cloudflare -- <instance> --migrate   # 有新 D1 SQL 时带 --migrate
npm run deploy:cloudflare -- <instance>             # 仅重新部署
```

本机先试用、不上云：见 [快速开始 · 本机启动](/zh/docs/quick-start/)。

## 配置模型（深入）

| 层级 | 说明 |
|------|------|
| `packages/*/wrangler.base.jsonc` | 已提交模板（无生产 `database_id`） |
| `packages/proxy/wrangler.jsonc` 等 | **生成产物**（gitignore），由 `gen:wrangler` 写入 |
| [`cloudflare-worker/example.env`](https://github.com/OctaFuse/octafuse-gateway/blob/main/cloudflare-worker/example.env) | **dev 演示** canonical 配置（可提交） |
| `cloudflare-worker/<instance>.env` | **生产/私有**（gitignore）；或仅用 Dashboard **Build variables** |

`gen-wrangler` 只读 **环境变量**（`process.env`），不关心来自 Dashboard 还是 `dotenv -e` 文件。

## 其它路径

| 路径 | 何时用 |
|------|--------|
| **本地开发** | 本机改代码、本地 D1（见 [快速开始](/zh/docs/quick-start/)） |
| **dev 演示** | 公共测试 `test-api.octafuse.dev`（`example.env` + CLI） |
| **Workers Builds** | Connect to Git 自动发版（D1 迁移仍需手动） |

### 远程部署后回到本地开发

执行过 `deploy:*` 或 `db:migrate:remote` 后，本机生成的 `wrangler.jsonc` 可能带有远程 `D1_DATABASE_ID`。回到本地开发前，请在没有导出 `D1_DATABASE_ID` 的 shell 中执行 `npm run gen:wrangler`，再重启 `dev:proxy` / `dev:admin`。

### Workers Builds 命令（可选）

**Root directory 留空**（monorepo 根）。`npm ci` 的 `postinstall` 会跑 `gen:wrangler`。

| Worker | Build command | Deploy command |
|--------|---------------|----------------|
| **Proxy** | `npm ci && npm run gen:wrangler` | `npm run deploy -w @octafuse/proxy` |
| **Admin** | `npm ci && npm run gen:wrangler && npm run build:cf -w @octafuse/admin` | `cd packages/admin && npx opennextjs-cloudflare deploy` |

<Aside type="caution" title="D1 迁移不在 CI">
	Git 流水线**不会**自动跑 D1 迁移。有新 SQL 时，用 `deploy:cloudflare -- <instance> --migrate` 或先 `db:migrate:remote`。
</Aside>

## Secrets 与下游

- **`ADMIN_PASSWORD`**：bootstrap 会写入；也可 `npx wrangler secret put ADMIN_PASSWORD --name <ADMIN_WORKER_NAME>`。
- 下游门户：`GATEWAY_URL`（Proxy）、`GATEWAY_MASTER_URL`（Admin）、`GATEWAY_MASTER_KEY`（D1 `system_config.MASTER_KEY`）。种子为 `sk-dev-admin-key` 时须在 Admin Config **轮换**。
