本页只做一件事：**在你自己的电脑上把 OctaFuse 跑起来**，并能打开管理台。更细的「怎么配供应商、路由、Key」见 **[使用指南](/zh/docs/guides/providers/)** 各章节。

<Aside type="note" title="命令所在仓库">
	以下命令在 **[octafuse-gateway](https://github.com/OctaFuse/octafuse-gateway)** 仓库**根目录**执行。
</Aside>

## 先决条件

- 已安装 **Docker Compose v2.20+**（需支持 `service_completed_successfully`）。
- 可访问 GitHub 以下载源码（或你的 fork）。

## 克隆并启动

```bash
git clone https://github.com/OctaFuse/octafuse-gateway.git
cd octafuse-gateway
docker compose -f docker/compose/quickstart.yml up --build
```

等待容器就绪后，新开终端执行：

```bash
curl -sS http://localhost:8787/health
```

若返回 JSON 且无报错，说明 **Proxy（推理入口）** 已监听 **`http://localhost:8787`**。

## 登录管理台

1. 浏览器打开 **`http://localhost:8789`**（**Admin**）。
2. 使用默认账号登录：**`admin` / `changeme`**（**仅限本地体验**；任何对外环境都必须改掉）。

## 可选：立刻打一条测试请求

在管理台里按使用指南配好 **Provider + 路由 + API Key** 后，可将下面命令里的密钥与 `model` 换成你的真实值：

```bash
curl -sS http://localhost:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"model":"your-route-model","messages":[{"role":"user","content":"Hello"}]}'
```

## 本地开发（不用 Docker 时）

在仓库根：`npm install`、`cp .env.example .env`，再 `npm run dev:proxy` / `npm run dev:admin` 与对应迁移命令。细节见 [Docker 部署](/zh/docs/deploy/docker/) 与 [GitHub 技术参考](/zh/docs/github-reference/)。
