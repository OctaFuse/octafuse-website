本页做一件事：按 **默认推荐的 Cloudflare 路径** 把 OctaFuse 跑起来，并能打开管理台。个人与小流量通常可在 Workers / D1 **免费额度**内完成部署与日常使用。

更细的「怎么配供应商、路由、Key」见 **[使用指南](/zh/docs/guides/providers/)**。不用 Cloudflare 时见 [Docker 部署](/zh/docs/deploy/docker/)。

<Aside type="note" title="命令所在仓库">
	以下命令在 **[octafuse-gateway](https://github.com/OctaFuse/octafuse-gateway)** 仓库**根目录**执行。
</Aside>

## 1. 本机启动（本地 D1）

### 先决条件

- **Node.js 20+**、npm。
- 无需 Cloudflare 账号。

### 克隆并启动

```bash
git clone https://github.com/OctaFuse/octafuse-gateway.git
cd octafuse-gateway
npm install
npm run db:migrate
npm run dev:proxy
```

另开终端：

```bash
npm run dev:admin
```

| 服务 | 地址 / 位置 |
|------|-------------|
| Proxy | `http://127.0.0.1:8787` |
| Admin | `http://127.0.0.1:8789` |
| 控制台登录 | `admin` / `admin`（本地默认；首次 `dev:admin` 会自动生成 `packages/admin/.dev.vars`） |
| 本地 D1 | `./.wrangler/state` |
| Admin API Bearer | `sk-dev-admin-key`（管理 API，不是网页密码） |

健康检查：

```bash
curl -sS http://127.0.0.1:8787/health
```

## 2. 部署到 Cloudflare

前置：Cloudflare 账号，本机已 `npx wrangler login`。

```bash
npx wrangler login
npm run bootstrap:cloudflare
```

完成后按终端提示核对 `GATEWAY_URL` / `GATEWAY_MASTER_URL`，并用 `GET $GATEWAY_URL/health` 验证。完整步骤与发版见 [Cloudflare Workers 部署](/zh/docs/deploy/cloudflare/)。

## 登录管理台

1. 本地打开 **`http://127.0.0.1:8789`**（上云后用 Admin URL）。
2. 本地默认账号：**`admin` / `admin`**（仅本机；可改 `packages/admin/.dev.vars`）。上云时使用 bootstrap 设置的 `ADMIN_PASSWORD`。Docker 本地一键体验默认是 `admin` / `changeme`，见 [Docker 部署](/zh/docs/deploy/docker/)。

## 可选：立刻打一条测试请求

在管理台里配好 **Provider + 路由 + API Key** 后，将密钥与 `model` 换成真实值：

```bash
curl -sS http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"model":"your-route-model","messages":[{"role":"user","content":"Hello"}]}'
```

上云后把主机换成你的 Proxy URL。
