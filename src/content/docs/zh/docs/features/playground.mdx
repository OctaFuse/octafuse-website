**Playground** 用于在 **不经过真实业务 Key** 的前提下，向某条 **模型路由** 发一条测试请求，快速确认「供应商地址、鉴权、上游模型名、自定义参数」是否正确。

<Aside type="tip" title="适用场景">
	改完路由、换上游、或新接入模型时，优先在 Playground 验证，再让业务流量切换。
</Aside>

## 如何使用

<Steps>
1. 打开 Admin 中的 **Playground** 入口。
1. 选择 **模型路由**（或等价选择器）：应能列出已配置的 active 路由。
1. 填写与协议匹配的请求体（例如 OpenAI 风格的 `messages`）。
1. **发送**：查看响应状态、上游错误信息、延迟等。
1. 若失败：回到 [供应商](/zh/docs/guides/providers/)、[路由](/zh/docs/guides/routing/) 逐项核对；仍不明时结合 [请求日志](/zh/docs/guides/request-logs/) 与主仓排障文档。
</Steps>

## 与真实调用的区别

Playground 为运维/开发试跑设计，**不会**像业务侧真实 Key 那样产生完全相同的计费与用量记录（行为以当前版本为准）。若要模拟「真实客户端 + 用户 Key」整条链路，请用 [Simulator](/zh/docs/features/simulator/)。
