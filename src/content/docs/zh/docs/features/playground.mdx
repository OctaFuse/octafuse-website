在左侧 **Inference → Playground** 打开。**Playground** 用来在 **不经过业务用户的 `sk-…` 计费路径** 的前提下，针对 **某一条已配置的路由** 向对应上游发一次试跑请求，快速确认供应商地址、密钥、上游模型名等是否正确。

试跑 **不会**写入与真实业务相同的 **请求日志** 流水，也 **不会**扣减用户侧预算，也 **不会**模拟「同一路由组内多条路由自动切换」的完整行为——后者请用真实 Proxy 调用或 [Simulator](/zh/docs/features/simulator/) 验证。

<Aside type="tip" title="适用场景">
	刚改完供应商或路由时，先用 Playground 确认能打通上游，再让业务流量切换。
</Aside>

## 如何使用

<Steps>
1. 打开 **Playground**。
1. 在路由下拉框中选择 **一条路由**（界面会用图标区分是否处于启用状态）。
1. 按当前协议（OpenAI / Anthropic / Gemini）编辑 JSON 请求体；可使用页面提供的模板起步。
1. 发送后查看响应状态、错误信息与正文；需要时留意响应头中的延迟等诊断信息。
</Steps>

## 与 Simulator 的区别

| 工具 | 典型用途 |
|------|----------|
| Playground | 运维/开发验证 **单条路由 → 上游** 是否配通 |
| [Simulator](/zh/docs/features/simulator/) | 用真实 **`sk-…`** 从浏览器打 **Proxy**，验证鉴权、选路、预算与日志整条链路 |
