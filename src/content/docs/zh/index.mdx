## 产品介绍

**OctaFuse** 是面向团队与企业场景的 **AI 网关**：为上层业务提供统一的模型调用入口与运营治理能力。你可以把它理解为组织内的「模型流量中台」——业务侧继续按熟悉的对话 / 补全习惯接入，而密钥、预算、路由、成本口径与审计留痕在网关层统一沉淀，便于协作、对账与合规。

适合希望在 **多供应商、多模型、多业务线** 并存时，仍能保持 **同一套治理口径与运营视角** 的团队。

## 为什么选择 OctaFuse

- **少重复建设**：不必每个系统各自对接供应商、维护密钥与限额；降低协作成本与长期维护负担。
- **兼容常见客户端习惯**：同时覆盖常见的 OpenAI、Anthropic、Gemini 等调用形态，迁移与共存更平滑。
- **成本与风险可见**：预算、用量与异常流量有统一视角；支持多视角成本口径，便于容量与对账讨论。
- **稳定性更可运营**：集中配置路由与优先级，在上游波动时更容易做切换与降级，而不是每个应用各自打补丁。

## 支持的部署模式

可按 **合规、延迟、运维习惯与数据落点** 选择部署方式；具体操作与命令见站内文档。

- **本地 / 内网 Docker 体验**：适合快速跑通、PoC 与内网联调，一条 Compose 拉起推理与管理界面。
- **Cloudflare 托管**：适合希望贴近全球边缘、降低自有机器运维面，并将元数据与配置托管在平台托管数据库上的场景。
- **自有基础设施自建**：适合对数据驻留、网络边界或与现有数据库运维体系一致性要求更高的组织（常见为 Postgres / MySQL 等）。

更多选型说明见 [文档首页](/zh/docs/) 中的线上部署章节，上手步骤见 [快速开始](/zh/docs/quick-start/)。开源协议为 **GNU AGPL v3**，许可全文见 [LICENSE](https://github.com/OctaFuse/octafuse-gateway/blob/main/LICENSE)。

## 适合谁

- 多产品线、多租户或平台型产品，希望 **统一治理模型调用** 的团队。
- 需要把 **成本、稳定性、合规留痕** 从业务代码里抽离出来，交给专门一层基础设施的工程与平台组。
- 希望在 **供应商与模型演进** 上保持主动权，而不是被分散集成绑死的组织。
