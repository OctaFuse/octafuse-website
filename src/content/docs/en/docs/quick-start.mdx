This page does **one thing**: **run OctaFuse on your machine** and open Admin. For “how to configure providers, routes, keys”, follow the **[User guide](/en/docs/guides/providers/)** sections.

<Aside type="note" title="Repository">
	Run commands from the **[octafuse-gateway](https://github.com/OctaFuse/octafuse-gateway)** repository **root**.
</Aside>

## Prerequisites

- **Docker Compose v2.20+** (needs `service_completed_successfully`).
- Network access to GitHub (or your fork).

## Clone and start

```bash
git clone https://github.com/OctaFuse/octafuse-gateway.git
cd octafuse-gateway
docker compose -f docker/compose/quickstart.yml up --build
```

When containers are ready, in another terminal:

```bash
curl -sS http://localhost:8787/health
```

Healthy JSON means **Proxy** is listening at **`http://localhost:8787`**.

## Sign in to Admin

1. Open **`http://localhost:8789`** (**Admin**) in a browser.
2. Sign in with **`admin` / `changeme`** (**local only**—change everywhere else).

## Optional: one inference smoke test

After Provider + route + API key exist in Admin, replace the key and `model` below:

```bash
curl -sS http://localhost:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"model":"your-route-model","messages":[{"role":"user","content":"Hello"}]}'
```

## Local dev without Docker

From repo root: `npm install`, `cp .env.example .env`, then `npm run dev:proxy` / `npm run dev:admin` plus migrations. See [Docker deployment](/en/docs/deploy/docker/) and [GitHub technical reference](/en/docs/github-reference/).
