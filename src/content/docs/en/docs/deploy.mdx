Common deployments are **Cloudflare Proxy Worker + Admin Pages + D1**, or **self-hosted Proxy + Admin + Postgres / MySQL**. This page is a high-level summary; step-by-step commands, secrets, and migrations stay in the main repository.

	For environment variables, **`metered_cost` / `standard_cost` / `charged_cost`**, migration versions, and **`/api/admin/*`** details, follow the main repository docs and **`packages/core/migrations-*`**.
</Aside>

<h2 id="deployment-modes">Common deployment patterns</h2>

Patterns described in [deployment.md](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/ops/deployment.md) include:

1. **Cloudflare fully managed (default)**: Proxy Worker + Admin Pages sharing **D1**.
2. **Hybrid**: **Proxy** on Node + **Postgres**, **Admin** still on Cloudflare + **D1**.
3. **Full self-hosted Postgres**: Proxy Node + Admin Node sharing **Postgres**.
4. **Full self-hosted MySQL**: same shape with **`DATABASE_DRIVER=mysql`** and **`migrations-mysql/`**.
5. **Docker + DB without Cloudflare**: pull **GHCR** images; migrations and compose in the Docker guide.

Local multi-env notes and D1 data dirs: [local-testing-environments.md](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/ops/local-testing-environments.md).

<h2 id="cloudflare">Cloudflare</h2>

Default Cloudflare path: two Workers (**Proxy**, **Admin**) sharing one **D1** database (binding **`DB`**).

- Overview: [docs/ops/deployment.md](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/ops/deployment.md)
- Cloudflare guide: [docs/ops/deployment-cloudflare.md](https://github.com/OctaFuse/octafuse-gateway/blob/main/docs/ops/deployment-cloudflare.md)

Typical endpoints:

- **Proxy**: **`GATEWAY_URL`** (e.g. `https://gateway.example.com`)
- **Admin**: **`GATEWAY_MASTER_URL`** (e.g. `https://gateway-admin.example.com`); admin API at **`GATEWAY_MASTER_URL`** + **`/api/admin/*`**
