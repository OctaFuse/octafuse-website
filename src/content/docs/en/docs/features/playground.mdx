Open **Inference → Playground**. **Playground** sends a **probe** for **one selected route** straight to its upstream, so you can verify URLs, credentials, and upstream model names **without** using a customer **`sk-…`** on the normal billing path.

It does **not** create the same **request log** stream as production traffic, does **not** debit user budgets, and does **not** fully mirror **multi-route failover** inside a route group — validate that with real Proxy calls or [Simulator](/en/docs/features/simulator/).

<Aside type="tip" title="When to use">
	After editing providers or routes, confirm upstream connectivity here before switching live traffic.
</Aside>

## How to use it

<Steps>
1. Open **Playground**.
1. Choose **one route** from the dropdown (icons indicate enabled vs disabled).
1. Edit the JSON body for the current protocol; start from the built-in templates if helpful.
1. Send and read status, errors, and body; check response headers for latency-style hints when troubleshooting.
</Steps>

## Versus Simulator

| Tool | Typical goal |
|------|----------------|
| Playground | Ops/dev: **single route → upstream** wiring check |
| [Simulator](/en/docs/features/simulator/) | Real **`sk-…`** from the browser to the **Proxy** — auth, routing, budget, logs |
