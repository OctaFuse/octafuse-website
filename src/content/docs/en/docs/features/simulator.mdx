Open **Inference → Simulator**. **Simulator** runs **in your browser** against the deployed **Proxy**: you enter the **Proxy base URL** and a **user `sk-…` key**, pick protocol and model options, and send — exercising **auth, routing, budget, and request logs** like a real client.

This differs from [Playground](/en/docs/features/playground/), which focuses on **Admin-assisted upstream probes** for a single route.

<Aside type="caution" title="Secrets">
	The **`sk-…` key lives in the browser** — prefer staging keys or server-side reproduction for production issues.
</Aside>

## How to use it

<Steps>
1. Open **Simulator**.
1. Enter the **Proxy base URL** (same entry point you give apps, including `https://` and port) and the **`sk-…`** key.
1. Choose protocol, model id, optional **route group** (must match [Routing](/en/docs/guides/routing/)). For image checks, use the Images / generations (or edits) entry — see [Image generation / edit](/en/docs/guides/images/).
1. Send, then confirm rows in [Request logs](/en/docs/guides/request-logs/) (including `billing_kind` / image counts).
1. If results differ from Playground: check **user budget**, **route group**, and **key status** (revoked, etc.).
</Steps>

The page may **remember** your last base URL and options **locally** in the browser for faster iteration — clear when switching machines or shared profiles.
