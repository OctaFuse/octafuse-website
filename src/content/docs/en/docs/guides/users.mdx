Open **User → Users**. A **user** is the account entity in the gateway: it owns API keys, carries **budget and billing period**, and appears across operational views. You can set email, optional external identifiers, and budget-related fields.

The **user detail** page (open a row from the list) typically shows the budget plan, related **API keys**, and shortcuts into **request** and **audit-style** history for that user (layout follows the current UI).

<Aside type="tip" title="Prerequisite">
	Finish [Quick start](/en/docs/quick-start/) and sign in to Admin.
</Aside>

## What to do in Admin

<Steps>
1. On **Users**, filter by email, external identity, status, and browse pages.
1. **Create a user**: fill email, optional external system and user id, budget caps/base, billing period, optional **metadata** (usually JSON for your own labels).
1. **Review a user**: open the detail view to check budget, keys, and logs.
1. **Issue keys**: create keys under [API keys](/en/docs/guides/api-keys/) for this user, or use the external-identity path in the key wizard.
</Steps>

## Portal integration

After signup, your backend can call the admin HTTP API to create users and keys — see [Integration](/en/docs/guides/integration/) and the [GitHub technical reference](/en/docs/github-reference/).
