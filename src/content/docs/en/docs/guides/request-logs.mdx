Open **User → Request Logs**. This view lists **real inference calls through the Proxy**: timestamps, keys, users, models, route groups, status, and more — useful for troubleshooting and reconciliation.

<Aside type="tip" title="Prerequisite">
	Traffic must hit the **Proxy**. [Playground](/en/docs/features/playground/) checks **do not** appear here; for production-like rows use a real key against the Proxy or [Simulator](/en/docs/features/simulator/).
</Aside>

## What to do in Admin

<Steps>
1. Open **Request Logs**.
1. Narrow results with time range, status, model, user email, API key, vendor, route group, protocol, and other filters available on the page.
1. Expand a row to inspect billing-audit snippets, request bodies, upstream bodies, and raw usage JSON (copy when provided).
1. When reading money and timestamps, align with your **system** timezone and billing currency settings in Admin.
</Steps>

## Billing display

Expanded panels may include charge-related JSON summaries; for exact formulas or finance reconciliation, follow your internal policy plus upstream docs linked from the [GitHub technical reference](/en/docs/github-reference/).
