**Request logs** capture Proxy traffic: timestamps, keys, users, models, route groups, status codes, latency (columns depend on release).

<Aside type="tip" title="Prerequisite">
	Some traffic has flowed through Proxy (test or real).
</Aside>

## In Admin

<Steps>
1. Open **Request logs** / **Usage** lists.
1. **Filter** by time, user, key, `model`, status.
1. **Debug failures** using error summaries; pair with upstream dashboards when needed.
1. **Export** if the UI supports CSV/JSON for offline analysis.
</Steps>
