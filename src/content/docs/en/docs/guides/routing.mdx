**Model routes** map a client **`model`** (and optional **route group**) to **provider + upstream model** with **priorities** for failover.

<Aside type="tip" title="Prerequisite">
	[Providers](/en/docs/guides/providers/) and [Models](/en/docs/guides/models/) should exist.
</Aside>

## In Admin

<Steps>
1. Open **Model routes** (or equivalent).
1. **Create a route**: pick **model**, **provider**, upstream model name, **route group** (often `default`, plus optional `free`, etc.), **priority**.
1. **Multiple routes**: same model + group can have ordered rows for failover.
1. **Status**: only **active** routes participate.
1. **Verify**: [Playground](/en/docs/features/playground/) or `curl` after issuing a key.
</Steps>
