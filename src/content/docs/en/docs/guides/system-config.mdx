Open **System → Config** (labeled **Config** in English). This screen groups global settings such as:

- **Master key** for server-to-admin API calls — must match **`GATEWAY_MASTER_KEY`** in your portal/backend.  
- **Business timezone** and **billing currency** — affect how times and money render across Admin; business timezone also drives route **Daily schedule** (time-of-day pricing) window evaluation.  
- **Alert webhooks** (e.g. Feishu / WeCom) — optional incident notifications.  
- **Custom entries** — advanced globals your team agrees on (names and meanings are operational contracts).

Sensitive values can be **shown or hidden** in the UI for screen-sharing.

<Aside type="caution" title="Security">
	After rotating the master key, update every script and backend that calls Admin. Use strong random values in production — not samples from docs.
</Aside>

## What to do in Admin

<Steps>
1. Open **Config**.
1. Verify master key, business timezone, and billing currency match your policy.
1. Configure or rotate alert URLs if you use them.
1. Add any other agreed global entries via **Add** (or the equivalent control) and save.
</Steps>

## Automation

Backends call Admin HTTP APIs on **`GATEWAY_MASTER_URL`** with **`Authorization: Bearer <same value as the master key in this screen>`**. Endpoint index: [GitHub technical reference](/en/docs/github-reference/).
