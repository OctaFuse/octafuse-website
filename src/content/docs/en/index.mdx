## Overview

**OctaFuse** is an open-source **AI Gateway** that consolidates model access scattered across vendors and plans into **one calling surface**.

- **Upstream**: OpenAI, Anthropic, Gemini, OpenRouter, coding / token / agent plans, plus Ollama, vLLM, and other self-hosted models—all wired in at the gateway layer.
- **Downstream**: clients need only **one Base URL** and **one API Key**, calling through familiar OpenAI / Anthropic / Gemini shapes without caring which provider handles the request.

Whether you are an individual juggling multiple plans and keys, or a team issuing separate keys for departments or customers with budgets and audit trails, OctaFuse runs on the same gateway stack.

## More use cases

- **AI app and agent builders**: keep product code on one protocol and Base URL; handle model swaps, canaries, degradation, and failover at the gateway.
- **API resellers and proxy operators**: connect multiple upstream providers and give each downstream customer its own key, budget, billing view, and logs.
- **Private and hybrid model stacks**: put Ollama, vLLM, or internal inference next to public cloud models behind the same gateway.
- **Labs, classrooms, and small teams**: admins configure models and quotas once; members share one key instead of maintaining vendor accounts individually.
- **Cost- and reliability-conscious teams**: compare spend and upstream health through routing, logs, and Analytics before picking defaults and backups.

## Why OctaFuse

Model supply keeps fragmenting—endpoints, API keys, quotas, invoices, and logs sit in different vendor consoles and tools. Switching models or debugging an incident means touching many places.

After reviewing many open-source and commercial options, we kept seeing the same pain points:

- **Narrow provider coverage**, making it awkward to mix public cloud, third-party inference, and self-hosted models.
- **Single-protocol surfaces**, forcing extra adapters when your stack already speaks Anthropic or Gemini.
- **Shallow billing and audit**, so it is hard to model per-route, per-user, or supply-side vs user-side costs.

OctaFuse addresses that as a gateway you can **own and evolve**:

- Wire in more upstream vendors—including plans and local / internal models—and expose multiple client protocols from one place.
- Define routing, billing lenses, and audit the way your workflow needs—whether you are consolidating personal keys or reconciling across teams.
- Integrate portals and apps through a stable **Admin API** with less coupling to vendor details.

## Deployment options

Pick based on **compliance, latency, data residency, and operations**:

- **Docker (laptop or private network)**: fastest path to try OctaFuse—great for PoCs, personal key consolidation, and internal debugging.
- **Cloudflare**: **Workers** for Proxy and Admin with **D1** for configuration and metadata.
- **Self-hosted**: stricter data boundaries or alignment with existing **Postgres / MySQL** operations.

See the production section on the [documentation home](/en/docs/) for trade-offs. Licensed under **GNU AGPL v3** — see [LICENSE](https://github.com/OctaFuse/octafuse-gateway/blob/main/LICENSE).
